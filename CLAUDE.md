# プロジェクトルール

## ドキュメント管理

- 実装・変更を行う前に必ず `docs/SPEC.md` を確認してから着手すること
- テストや動作確認の前にも `docs/SPEC.md` で仕様を確認すること
- 実装が完了したら `docs/TASKS.md` の該当チェックボックスを更新すること
- 機能追加・技術変更があった場合は `docs/SPEC.md` も同時に更新すること

## テスト

- テストが失敗した場合は**実装コードを修正**して対応すること
- 以下の行為は**禁止**：
  - テストコード・テスト仕様自体の変更
  - 期待値やアサーションの書き換えによる合格
  - テストケースの削除・スキップ（`skip` / `todo` / `xit` 等）による失敗の回避
- テスト仕様の変更が本当に必要な場合は必ずユーザーに確認を取ってから行うこと

## ドキュメント構成

```
docs/
├── SPEC.md    # 仕様書（技術スタック・決定事項・機能・スキーマ等）
└── TASKS.md   # タスク進捗管理
```
